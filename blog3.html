People Sing Very Meaningful Songs Altogether

Say What?

That may be a little kumbaya for your taste, but it was the first thing that popped into my head when I was trying to develop a mnemonic to remember the opening line for a java program.

people sing very meaningful songs altogether
public static void main String[] args

When I first started to learn java, I never thought I'd remember it. With the mnemonic, it's a piece of cake.

public static . . . main

Parts of the opening are a little more complicated. I know that this is the main program to run and that it must be declared public and static. I presume one could also have programs (methods) which are private and dynamic, but I haven't advanced that far yet. In any case, the main program should always be public and static.

void vs int

The word void in the main program just means that it doesn't return a value. You can have other methods, called by your main program, which return a value. See the following example. I like to force the user to give me an integer if an integer is expected. In this case, I want the method to return an integer. So instead of public static void, I will use public static int before the name of my method. This method takes one parameter, a String with the variable name prompt.

public static int getCleanInt(String prompt) {

Scanner scanner = new Scanner(System.in);

System.out.print(prompt);

while(!scanner.hasNextInt()){

System.out.print("Please enter only an integer: ");

scanner.next();

}

return scanner.nextInt();

}

In the main program, I might call this method to fill a variable.

int age;

age = getCleanInt("Enter your age: ");

There's nothing in my method preventing the user from entering a negative number. To solve this problem, I might have another method, which calls the getCleanInt() method.

public static int getReasonableAge(int num) {

while( (num < 0) || (num > 130) ) {

num = getCleanInt("Enter your age: ");

}

return num;

}

Then I would do this in main program

int age;

age = getReasonableAge(-1);

By calling my method with a number out of range, I start the process of getting a "reasonable" number, which, by combinging these two methods, is an integer between 0 and 130.
