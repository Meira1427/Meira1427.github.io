If Else Switch

if/else

If you have one condition to check, an if/else would be your best choice. Example:

if (answer == correctAnswer) {

[some code here]

}

else {

[some code here]

}

You would also choose an if/else if you were testing for a range of values, and performing the same code for every value in the range. Here again, you have two possible branches of code.

if ((answer > 0) && (answer < 100)) {

[some code here]

}

else {

[some code here]

}

if/else if

If you are creating some branches in your code and you are looking for a few different potential answers or several ranges of answers, an if/else if is the most useful. An example:

if (answer < 0) {

[some code here]

}

else if (answer > 100) {

[some code here]

}

else {

[some code here]

}

switch

If you have multiple possible outcomes and each answer is unique (i.e. not a range of values), the code might look cleaner with a switch statement. For example:

switch(birthMonth) {

case "Jan":

[some code here]

break;

case "Feb":

[some code here]

break;

case "Mar":

[some code here]

break;

[etc with the rest of the months]

default:

[some code here]

}

In this example, an if statement with eleven else if branches would be less useful than a switch. Each possible instance in a switch needs a break at the end of the code. Overall, a switch is tidy and compact when you will be choosing from one of a series of unique values. If you will be working with ranges of values or if the answer will be one of only a few choices, you will be better off with an if/else if/else.
