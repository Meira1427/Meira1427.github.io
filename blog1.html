Assign vs Equals vs .equals

One Equals Sign "="

In Java, you use a single equal sign to assign a value to a variable. The value on the right is assigned to the variable on the left. For example:

count = 0;

This puts the value of zero into a variable called count. I am assuming the variable was already initialized. If not, the statement would have been:

int count = 0;

Later in the program, you might see a new assignment for this variable:

count = count + 1;

or

count++;

a shortcut for count + 1.

In these last two cases, the program will take whatever value is currently stored in count and increment it by one.

Two Equals Signs "=="

If you want to compare two numeric or boolean values, you must use "==". Some examples:

5 == 5

if (count == 10) {

while (playing == true) {

if (pin == 1234) {

In these cases, the program will check the condition, then continue with the block of code (not specified in example) if true. If false, it will continue to next block of code.

.equals

Be careful. If you want to compare two string values, you must use the .equals() method instead of "==". Some examples:

if (password.equals("JavaRocks")) {

while (playAgain.equals("yes") || playAgain.equals("y") ) {

if (userAnswer.equals(correctAnswer)) {

In all of these cases, Java is comparing the string in the variable on the left to the item within the parentheses after .equals. If they match, it will continue with the block of code (not specified in example). If they do not match, the program will move on to the next block of code. You cannot use "==" when when comparing strings. For example, do not do this:

if(password == "JavaRocks") {

if you want your program to work as intended.

Overall, pretty easy to keep apart, as long as you don't forget and try to use "==" with strings.
